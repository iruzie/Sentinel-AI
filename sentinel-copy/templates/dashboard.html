Sentinel-AI — Intelligent Web Application Firewall

Sentinel-AI is a real-time, ML-powered Web Application Firewall (WAF) built on a three-tier architecture: an NGINX edge proxy, a scikit-learn inference microservice, and a FastAPI analytics dashboard. It intercepts every HTTP request before it reaches your origin server, classifies payloads using trained ML models and regex blacklists, and logs every decision for forensic review.

NGINX map · user-agent
Rule-based

Proxy Cache
NGINX · sentinel_cache zone
Passive

Cache TTL 60 seconds
🔐 HTTP Security Headers (NGINX)
X-Frame-Options
DENY — Prevents the page from being embedded in iframes. Mitigates clickjacking attacks.
X-XSS-Protection
1; mode=block — Enables the browser's built-in XSS filter and blocks rendering if an attack is detected.
X-Content-Type-Options
nosniff — Stops browsers from MIME-type sniffing, preventing content injection via malformed file types.
Referrer-Policy
strict-origin-when-cross-origin — Limits referrer information shared cross-origin to only the origin URL.
Content-Security-Policy
default-src 'self' — Restricts all resource loading to the same origin, blocking inline scripts and external assets.
X-Cache
$upstream_cache_status — Exposes cache hit/miss status (HIT, MISS, BYPASS) for debugging proxy cache performance.
⚙️ NGINX Configuration Highlights
worker_processes auto; # Auto-scale to CPU cores # Rate limiting: 2 req/s per IP, burst of 3 (nodelay = instant reject) limit_req_zone $binary_remote_addr zone=cdn_limit:10m rate=2r/s; limit_req_status 429; # Proxy cache: 10 MB keys zone, 100 MB max disk, 60 s inactive TTL proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=sentinel_cache:10m max_size=100m inactive=60s use_temp_path=off; # Bot detection via User-Agent map (blocks before ML engine is called) map $http_user_agent $is_bot { ~*curl | ~*wget | ~*sqlmap | ~*nikto | ~*nmap | ~*python | ~*httpie → 1; } # ML auth sub-request (internal – every passing request is inspected) auth_request /ml_check; # proxies to ml:5000/predict (POST) # GZip compression (text/plain, CSS, JSON, JS ≥ 256 bytes) gzip on; gzip_min_length 256;
✨ Key Features
  • 🧠 Layered ML defence — Three independent scikit-learn classifiers (Global, XSS, SQLi) with tunable probability thresholds run in sequence on every request.
  • 📜 Regex blacklist pre-filter — 12 battle-tested SQL injection patterns are checked first, ensuring zero-latency blocking of the most common attacks.
  • 🤖 NGINX bot fingerprinting — Seven scanner/tool User-Agent signatures are matched at the edge before any Python code runs.
  • ⏱ Per-IP rate limiting — Configurable request rate (default 2 req/s, burst 3) with HTTP 429 responses to mitigate brute-force and DDoS.
  • 🗂 Proxy caching — 60-second response cache reduces origin load and provides stale-while-revalidate resilience during ML service restarts.
  • 📊 Real-time analytics dashboard — Live charts, threat categorisation, top-IP tracking and live log streaming with 5-second auto-refresh.
  • 🚫 Dynamic IP blacklist — Block/unblock IPs with one click via the REST API — changes are persisted to blacklist.json.
  • 📂 Forensic log export — Download filtered security logs as CSV or formatted PDF reports (powered by ReportLab) for compliance and auditing.
  • 🔐 Hardened HTTP headers — Five security response headers configured at the NGINX layer: CSP, X-Frame-Options, XSS-Protection, MIME sniff guard, Referrer-Policy.
  • 📦 Fully containerised — Three-service Docker Compose stack (origin, edge, ml) with shared Docker volumes for logs and quarantine events.
🚀 How to Run
Option A — Docker Compose (full stack)
docker compose up --build

Edge WAF runs on http://localhost:8080 · ML engine on port 5000

Option B — Local development (dashboard only)
source .venv/bin/activate uvicorn main:app --reload # Dashboard → http://localhost:8000 # In a separate terminal: cd ml && uvicorn app:app --reload --port 5000 # ML engine
